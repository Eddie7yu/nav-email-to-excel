# 净值邮件写入 Excel Skill

这是一个面向中文用户和本地 AI Agent 的公开 Skill：用户只需在本机输入一次邮箱授权码，AI 就能发现净值发件人、匹配产品与工作表；没有现成工作簿时还会创建完全脱敏的标准净值表。完成第一次预览验收后，系统自动备份和更新 Excel。

发件人邮箱、产品代码、主题规律、列位置和邮件到达时间都由 AI 从本地邮箱与工作簿中发现，不要求用户手工整理。只有历史证据无法唯一确定业务口径时，AI 才询问用户。

## v2.0 的理念：成熟路径，不是封闭答案

这个仓库经过多轮真实部署反馈，已经把首次安装、邮箱发现、批量产品路由、HTML/附件解析、工作表接管、零新增上线、BAT 闪退、预览批准、备份和定时更新等常见问题沉淀成可复用的成功路径。但外部邮箱、附件和工作簿格式不会被任何手册穷尽；这些方案是 Agent 应优先参考的经验，不是限制它只能照固定步骤行动。

遇到文档没有覆盖的情况，Agent 可以在本机自主检查证据、尝试不同解析方法、对工作簿副本试验、编写临时诊断脚本或增加受信任本地解析器。探索方法可以灵活，交付结果必须可证明：真实资料不外泄，产品和数值无歧义，第一次结果由用户验收，已有工作簿保持原有结构，正式写入经过验证、备份、临时副本和原子替换，异常时正式表不变。简而言之：**已知问题复用成熟方案，未知问题允许 Agent 发挥；自由发生在解题过程，安全收口发生在正式交付。**

仓库中的短路径、扫描批次、历史接管范围、表头语义和解析方式都是跨机器可调整的默认值，不是把某次测试电脑的环境写成安装门槛。普通用户仍只表达业务目标；Agent 根据目标电脑、邮箱和工作簿证据选择参数或本地适配，并在同一套预览与验证链路中收口。

## 它解决什么问题

- 从纯文本/HTML 表格正文或 Excel、CSV、文本型 PDF 附件中提取净值。
- 固定标签/值 XLSX 配套 PDF 这类供应商专有结构提供完全虚构的严格本地解析器示例，覆盖身份、主题/附件日期、累计净值、附件数量和同日冲突校验，但默认不启用也不扩大通用解析器。
- 可先只读 `From/Subject` 邮件头收窄候选，再仅下载精确发件人和主题范围内的完整邮件，自动提出“发件人/产品 → 工作表”路由；已有表保持原样，没有表则从内置脱敏模板新建。
- 内置周度/日度、无指数/有指数四类产品页和可见指数来源页，一只产品一个工作表；自动维护日收益、已完成自然周的周收益、指数收益和超额。
- 支持两种低门槛新增产品方式：原样接管用户已建 Sheet，或照受管 Sheet 完整复制格式、清空参考产品内容后接管；都自动匹配邮箱并只需再验收一次预览。
- 接管带基准/超额列的已有 Sheet 时，暂时无法证明来源也能先生成审查预览；预览会留空未确认的新增基准/超额并禁止提交，AI 补齐可靠来源、重新预览后再上线。空白预留基准列不会误拦截。
- 汇总模式通常用两个历史日期完成无警告核验，但首次预览只需产品名称或产品代码任一唯一对应，以及至少一个真实邮件日期；零重叠或只有一个重叠日期会显示醒目警告，由 Agent 和用户检查后再批准。若表内只有一条“产品标识+日期、净值为空”的预留数据行，程序也可从一条真实邮件安全冷启动并保留原汇总行。
- 接管已有历史的 Sheet 时默认显示并保存“原表最后日期 + 1 天”的受管边界，以及只读 `last_existing_point` 重叠锚点，从表尾安全续接；需要补历史时，Agent 可切换到受监督的邮箱历史范围或自定义起点。该边界只限制候选数据写入，不截断原表收益、累计和基准公式的完整历史；最后既有点不会被新增或覆盖。任何历史新增或修复仍必须进入受保护预览并批准。
- 只追加工作簿尾部的新日期；发现历史内部缺口、重复日期或同日冲突时停止。
- 邮箱提供真实历史，现有 Excel 决定落表结构和数据频率：运行时会从既有日期规律或明确表头识别日度/周度行，并拒绝与模板冲突的强制设置；周度模板不会被日度邮件改造成日度空壳。若当前只有未完成自然周的真实邮件，身份和工作簿尾部可核实即可生成零新增基线，本周仍不写；原历史、公式、汇总行、基准和样式必须保留。
- 第一次配置生成一份可检查的本地预览；用户批准一次后，后续不再要求重复预览。
- 自动更新仍绑定配置、正式工作簿和内部临时验收副本的哈希，写入前备份，结束后删除临时副本。
- Windows 下通过 Excel/WPS COM 写入临时副本，验证后备份并原子替换正式文件。
- Windows 定时任务在首次批准后自动验证、备份和写入；异常时停止并保持原表不变。

## 为什么能少踩很多坑

仓库把此前最容易反复出错的环节做成了已验证实现、诊断参考和回归测试，包括：

- 依赖版本锁定、隔离虚拟环境、Windows 短路径预检、固定运行时 Python，以及 BAT 留窗和特殊字符路径回归；
- Windows 中文筛选参数支持 UTF-8 参数文件；CLI 会拒绝明显乱码并在本地报告区分“编码已核验”与真实零命中，避免把终端代码页问题误判成邮箱无邮件；
- 头部优先候选扫描、选择式完整邮件下载、发件人二次核对、批量邮件只选择已配置产品、产品代码精确匹配、停更路由显式暂停和陈旧数据拦截；
- 单主题短窗口分批、实时邮件/附件进度、软时间预算和 UID 断点恢复；主题唯一代码可显式绑定无代码附件行，份额后缀只能经确认保存精确别名；
- 读取中短暂 IMAP 断线会在同一只读扫描内最多自动重连两次，并从当前 UID 继续；只有 UIDVALIDITY 未变化才复用当前范围，持续断线或邮箱身份变化仍安全停止且不采用不完整结果；
- 多路由失败报告会给出最小重叠路由对、冲突日期和逐附件脱敏来源指纹，并允许按 Sheet 运行只读发现或不可提交的局部预览；最终批准前仍统一执行完整预览；
- `products status/sync` 原生输出逐 Sheet 覆盖矩阵；“全部产品”表示每页都有活动、暂停、参考、明确排除、邮箱无证据、需本地解析器或业务待确认等可审计结论，不会强迫无证据或已结束页面建立路由；
- 完整发现成功后保存本地指纹快照；只调整工作簿列、基准或公式时，可先轻量复核邮件头、UID、大小、路由和解析器未变，再复用解析结果生成新的完整预览，避免反复下载全部附件；
- 正式净值主题前缀和产品代码共同约束路由；虚拟估算及不含明确净值语义的交易确认、月报/季报在下载前排除并按受控原因计数，同时含净值语义的主题仍继续解析，避免把报告 PDF 误当成解析器缺口或把有效通知静默排除；
- 同日不同值默认停止；只有同产品同日期、后发且主题明确标注更正/以此为准、单封内部唯一的正式更正通知才可替代原值，并在报告中保留脱敏的替代来源链；
- 历史校验会发现代码列递增伪代码、缺失净值和孤立的单位/累计价差断点；邮件能证明旧表错误时也只能先生成修复预览差异，用户批准后才按备份与 COM 临时副本保护应用；
- 累计净值口径、序列重置、日/周收益和基准对齐；
- 未来日期、异常净值跳变、重复写入和历史缺口保护；
- 严格汇总模式、带预留行的汇总冷启动与简单追加模式的结构隔离，汇总公式失败关闭、格式、工作表结构、预览完整性及正式文件不变性检查；
- 本地密钥、日志、预览、备份的隔离与公开仓库脱敏审计。
- 内置模板逐项检查 XLSX 内部 XML，拒绝真实产品、净值、日期、邮箱、费率、作者、链接、宏、批注、隐藏内容和外部连接；其他 Excel 文件仍禁止进入 Git。

它不能保证从未见过的邮件格式都自动兼容，也不要求 Agent 把新问题强行套进旧案例。AI 可以利用邮箱历史、工作簿副本、临时诊断脚本、现有工具和本地解析器探索；只有业务证据仍无法唯一判断时才向用户说明问题。如需持久支持特殊格式，可使用运行目录 `app/parsers/` 中的扩展点并用完全脱敏样例回归，绝不能从邮件或附件加载代码。

第一次部署可先阅读面向非程序员的[中文使用流程](使用流程.md)。

## 适用范围与限制

- 邮箱：IMAP over SSL，使用应用专用密码或授权码；支持标准 IMAP，以及需要登录后发送 ID 的 163、126、yeah.net 和网易企业邮主机；暂不支持仅限 OAuth 的登录。常见服务商地址、认证限制和核验方法见 [IMAP 服务商参考](references/email-providers.md)。
- 附件：Excel、CSV 和文本型 PDF；扫描 PDF 暂不提供 OCR。
- 预览：Windows、Linux、macOS 均可运行 Python 流程；持续集成覆盖 Windows、Linux，并对 macOS 做预览 smoke test。
- 正式写入：仅支持 Windows，并要求 Microsoft Excel 或 WPS 表格的 COM 接口可用。
- 定时任务：仅支持 Windows 登录态；首次批准后自动备份并写入正式工作簿，无需用户逐次检查预览。
- 工作簿：已有 `.xlsx`/`.xlsm` 会被完整保留，绝不套模板、改表头或重排；`summary` 与 `append` 规则不变。没有工作簿时可登记尚不存在的 `.xlsx` 路径，AI 根据邮件证据选择日/周频率以及是否有基准，再运行 `workbook init-template` 创建。程序生成的模板允许一个真实邮件日期带警告冷启动，两个已核实日期后恢复严格验证；任何已有目标文件都拒绝覆盖。

```powershell
git clone https://github.com/Eddie7yu/nav-email-to-excel "$env:USERPROFILE\.codex\skills\nav-email-to-excel"
codex
```

然后输入：

```text

```

Claude Code、Cursor 及其他本地 AI 的安装方法和完整提示词见 [交给其他本地 AI 使用](references/portable-use.md)。AI 应优先调用仓库内已验证脚本；遇到未覆盖格式时可以自行诊断和扩展解析，只要最终重新进入受保护的预览、批准和写入链路。

## 先做一次完全离线演练

离线演练用于验收目标电脑，不会读取真实 `app/config.json`、邮箱、密钥或正式工作簿。它使用虚构邮件和虚构 Excel，实际走过邮件发现、两个历史日期核验、预览、公式和保护逻辑，从而把“电脑环境问题”和“真实业务配置问题”分开。

AI 进入已经创建的运行目录的 `app/`，生成虚构预览：

```powershell
cd D:\data\净值自动化\app
.\.venv\Scripts\python.exe navctl.py demo prepare
```

命令会分阶段报告结果并返回 `run_id`。AI 检查虚构结果后，在 Windows 上继续演练 Excel/WPS COM 写入：

```powershell
.\.venv\Scripts\python.exe navctl.py demo commit --run-id <run_id> --yes-reviewed-preview
```

完成后清理该次虚构演练：

```powershell
.\.venv\Scripts\python.exe navctl.py demo remove --run-id <run_id>
```

`demo commit` 不会触碰正式工作簿。macOS/Linux 可以完成 `demo prepare`，但不能演练仅限 Windows 的 COM 正式写入。

## 首次部署的推荐成功路径

1. AI 完整阅读 [SKILL.md](SKILL.md) 和[配置说明](references/configuration.md)，默认在工作簿旁创建独立的 `净值自动化/` 运行环境并完成离线演练。
2. AI 获取已有工作簿路径；如果没有，则获取希望新建的 `.xlsx` 路径。再获取收件邮箱账号；IMAP 设置优先从现有客户端或服务商资料中核实。
3. AI 运行 `navctl.py secret launch` 主动弹出可见窗口；用户只需在该窗口隐藏粘贴一次授权码，AI 随后用 `secret status` 检查，不要求用户输入终端命令。
4. AI 优先用 `navctl.py propose --headers-only` 读取最小邮件头，再按“稳定主题 + 精确发件人 + 短窗口”逐产品解析；主题不确定时仍保留发件人。默认批次带进度、软时间预算和可恢复 UID 游标，未完成时运行 `propose --resume`，无范围完整扫描必须显式启用。已有表同时参与结构判断。随后给出运行时间建议，并询问用户希望每周一次、工作日每天或哪些星期、几点自动更新。
5. 没有现成工作簿时，AI 核实日/周频率和基准后运行 `navctl.py workbook init-template`；有现成表时跳过，绝不套模板。多产品已有表先用 `products adopt --inspect-only` 分组并检查列角色：普通续接默认从表尾开始，需要补历史时由 Agent 内部切换到受监督的邮箱历史范围。证据完整的产品先进入可提交预览，基准待核实产品单独审查，不拖住前者。权威指数能唯一映射时自动配置；工作簿自定义基准名称无法由本地或公开证据唯一映射时才询问用户，不猜相似指数。份额代码后缀只列为待确认别名，核实后保存精确别名，绝不模糊接管。然后完成 `doctor` 并直接运行第一次 `preview`，不重复串行执行 `discover`、`validate`。
6. 用户检查第一次预览并批准。AI 执行 `commit --yes-reviewed-preview`：有新增时自动备份并写入；零新增时检查中文基线报告，只批准当前配置而不修改工作簿。两种情况都能继续上线自动更新。
7. AI 按用户确认的频率和时间安装计划任务。定时任务是 Windows 正式部署的必做步骤；以后自动验证、备份和写入，没有新数据不改文件，异常则停下并记录原因。

部署后，普通用户新增产品只需要告诉 AI 两件事之一：①“我已经建好这个 Sheet，请接管”，AI 默认使用 `products adopt` 自动识别表头、频率和邮箱产品；②“这个新产品格式照某个现有 Sheet”，AI 默认使用 `products clone` 通过 Excel/WPS 完整复制格式、清空参考产品内容并接管。这是简化用户操作的自然语言入口，不是限制 Agent 处理特殊工作簿的内部手段。两种方式都不需要重新部署、重新输入授权码或重装定时任务，只需检查一次新产品预览。完整边界见 [SKILL.md 的新增产品章节](SKILL.md#普通用户新增产品的两种自然语言入口)。

需要夏普比率、最大回撤、年化收益等专业指标时，先由本工具生成经核验的净值序列，再让 AI 在未托管分析工作表或单独文件中按一致口径计算并说明假设。详见[配置说明中的“派生专业指标”](references/configuration.md#派生专业指标)。

Windows 默认把运行目录放在工作簿同级的 `净值自动化/`，一份工作簿对应一套配置、入口、备份、日志和计划任务。引导程序把 116 个字符定义为当前支持上限，为内部 `app/` 和深层依赖保留空间，并会在创建文件前拒绝更长路径；默认位置过长、不可写或已有同名目录时，再显式选择 `D:\nav-runtime` 之类的新短本地路径。这是本工具的可靠部署边界，不是 Windows 的通用路径上限。

## 交付给普通用户的目录

引导完成后的运行目录已经分成“普通用户入口”和“内部程序”两层：

```text
D:\data\
├─ nav.xlsx
└─ 净值自动化\
   ├─ 使用说明.txt
   ├─ 首次授权.bat
   ├─ 查看状态.bat
   ├─ 手动更新.bat
   ├─ .gitignore        # 防止本地资料被误提交
   ├─ previews\
   ├─ backups\
   ├─ logs\
   └─ app\              # Python、配置、状态、解析器和虚拟环境
```

普通用户日常只接触中文入口和三个结果目录，不需要进入 `app/`，也无需修改 `.gitignore`。手动更新与计划任务共用内部更新入口，因此首次尚未批准、配置错误等失败也会留下同格式的本地日志。AI 执行命令、维护配置或排错时才进入 `app/`；探索期间可以把临时脚本和中间结果放在 `app/diagnostics/`，但不得堆在用户根目录，并应在交付前清理。详细边界和交付检查见[运行目录分层规范](references/runtime-layout.md)。

已安装自动更新后，AI 可用以下命令查看最近一次运行、写入行数和备份路径：

```powershell
cd D:\data\净值自动化\app
.\.venv\Scripts\python.exe navctl.py schedule status
```

普通用户也可以直接双击根目录的 `查看状态.bat`。
状态会明确显示 Windows 本地时区和计划任务的本地运行时间，避免把任务时间误读成 UTC。

## 升级已部署的运行环境

不要在旧运行目录中覆盖脚本、替换 `.venv` 或直接复制新版模板。安全升级采用“新建运行目录、迁移配置、重新验收”，旧目录在验收完成前保留为回退副本：

1. 进入旧运行目录的 `app/` 执行 `navctl.py schedule remove`，避免新旧任务同时运行。
2. 更新干净的 Skill 仓库，为同一正式工作簿显式指定一个新的短运行目录重新运行 `scripts/bootstrap.py`（例如 `--destination "D:\nav-runtime-upgrade"`）；默认的同级 `净值自动化/` 仍在使用时不会被覆盖。
3. 以新生成的 `app/config.json` 为基础，让 AI 迁移并复核 `routes`、`sheet_reviews`、`column_overrides`、`style`、`schedule`、`validation` 和 `retention`。不要用旧文件整体覆盖，以免带回旧的结构版本或 `runtime_id`。
4. 如有本地 `app/parsers/`，逐个审阅后复制到新运行目录的同名位置；不要迁移 `app/.venv`、密钥、`plan.json`、`run.lock`、预览、日志或定时状态文件。
5. 在新目录重新执行 `secret launch` 并由 AI 用 `secret status` 复查，再完成离线演练、验证和一次预览。新预览确认后重新安装自动更新任务并停用旧目录；不要迁移 `automation-approval.json`。

当前没有 `navctl upgrade`，这是有意保留的失败关闭设计：配置结构或运行时依赖变化时，升级必须经过重新验收，不能静默改动正式环境。

## 公开仓库与本地数据的边界

公开仓库只包含通用 Skill、确定性脚本、虚构测试数据，以及从零生成并通过内部 XML 审计的 `assets/workbook-templates/nav-standard-cn.xlsx`。它不来自任何真实工作簿。真实邮箱、发件人、产品、工作簿、路径、日志、预览、备份、配置和凭据必须留在单独的本地运行目录，绝不能提交或分享。

发布前应运行：

```powershell
python -X utf8 scripts/privacy_audit.py
python -X utf8 scripts/selftest.py
python -X utf8 scripts/selftest.py --com
python -X utf8 scripts/package_check.py
```

普通 `selftest.py` 适用于所有平台；`--com` 会在 Windows 上额外创建临时虚构工作簿，真实调用 Excel/WPS COM，复核公式缓存值后自动清理。测试输出会逐阶段说明检查内容，全程不使用真实资料。
