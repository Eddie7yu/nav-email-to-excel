# 使用流程：从零到每周自动更新净值表

> 这份文档写给准备在自己电脑上用这套工具的你。你不需要会编程——所有命令由你的
> AI 助手（Codex / Claude Code / Cursor）执行，你只负责三件事：**提供信息、检查
> 预览、给出批准**。你的邮箱授权码和全部数据只留在你自己的电脑上。

## 它帮你做什么

从邮箱里的净值邮件提取数据 → 与工作簿历史数据核对 → 生成预览副本给你检查 →
你批准后才写入正式 Excel 表格，原有公式、格式、图表保持不变，写入前自动备份。

它刻意"胆小"：数据冲突、日期缺口、格式变化等任何拿不准的情况都会停下来说明
原因，而不是替你猜。**初次配置时被它拦住是正常现象**，跟着提示逐项解决即可。

## 第 0 步：准备清单（开始前先备齐）

软硬件：

- [ ] Windows 10/11 电脑（正式写入只支持 Windows；Mac/Linux 只能做到预览）
- [ ] Microsoft Excel 或 WPS 表格（任一）
- [ ] Python 3.11–3.14（python.org 下载，安装时勾选 "Add python.exe to PATH"）
- [ ] Git（git-scm.com，一路默认安装即可）
- [ ] AI 编程助手：Codex、Claude Code 或 Cursor 任一

信息（不确定的先问清楚再开始）：

- [ ] 正式净值工作簿的完整路径（`.xlsx` / `.xlsm`；目标工作表已存在，可以有历史数据，也可以是准备初始化的空白表）
- [ ] 收净值邮件的邮箱账号和 IMAP 服务器地址（如 `imap.qq.com`、`imap.exmail.qq.com`）；
      不知道地址时让 AI 按仓库内 `references/email-providers.md` 核验，不要自己猜
- [ ] 该邮箱的 **IMAP 授权码**：在邮箱网页版设置里开启 IMAP 服务并生成，不是登录
      密码。★ 拿到后不要发给任何人、任何聊天窗口，第 3 步会用专门方式输入
- [ ] 哪些发件人的邮件要自动处理；每个发件人对应的产品名称或代码、写入哪张工作表
- [ ] 每个产品的口径：累计净值邮件里有没有、收益按日还是按周、基准指数在工作簿
      的哪张表里

> 邮箱兼容性：支持标准 IMAP over SSL。v1.2.1 起，163、126、yeah.net 和网易
> 企业邮主机会在登录后自动发送所需的 IMAP ID；仅限 OAuth、没有授权码登录方式的
> 邮箱仍暂不支持。

## 第 1 步：把 skill 装给你的 AI

在 PowerShell 里执行（三选一）：

```powershell
# Codex
git clone https://github.com/Eddie7yu/nav-email-to-excel "$env:USERPROFILE\.codex\skills\nav-email-to-excel"
codex

# Claude Code
git clone https://github.com/Eddie7yu/nav-email-to-excel "$env:USERPROFILE\.claude\skills\nav-email-to-excel"
claude
```

Cursor 用户：克隆到任意本地目录，让 Agent 完整阅读其中的 `SKILL.md`（详见仓库内
`references/portable-use.md`）。

## 第 2 步：对 AI 说这段话（直接复制，替换尖括号内容）

```text
请使用 nav-email-to-excel skill，完整阅读 SKILL.md 和 references/configuration.md，
为我的工作簿部署一套预览优先的净值邮件自动化：<你的工作簿完整路径>

全程从只读开始。未经我明确同意，不得写入正式工作簿、发送邮件或安装定时任务。
不要让我把邮箱授权码发到聊天里；需要密钥时，告诉我如何自己在本机隐藏输入。
产品、日期或数值有歧义时必须停下来问我。先生成预览和验收报告，我检查并批准后
才能正式写入。
```

## 第 3 步：跟着 AI 走完部署（你在每一步做什么）

| 阶段 | AI 做什么 | 你做什么 |
|---|---|---|
| 1. 创建运行目录 | 运行引导脚本，装好隔离的运行环境 | 提供安装目录——**用 `D:\nav-runtime` 这类短路径**（不要放云盘或很深的文件夹） |
| 2. 离线演练 | 用完全虚构的邮件和表格把全流程走一遍 | 看到全部 PASS 即电脑环境合格；这一步不碰你的真实邮箱和表格 |
| 3. 保存授权码 | 只告诉你命令，不经手密码 | ★ 你亲自在终端执行 `.\.venv\Scripts\python.exe navctl.py secret set`，粘贴授权码（屏幕不显示），回车 |
| 4. 配置路由 | 读取工作簿结构，逐条与你确认“发件人 → 产品名称/代码 → 工作表”、严格汇总或简单追加模式及各项口径 | 逐条确认；空白/简单表只能显式选择追加模式，不能让 AI 因验证失败自动降级；不确定就说不确定，**不要让它猜** |
| 5. 四步验证 | 依次运行 doctor / discover / validate / preview | 听 AI 汇报结果；有拦截按提示解决 |
| 6. ★ 检查预览 | 告知预览文件路径（运行目录的 `previews\` 里） | **亲自打开预览 Excel**：核对新增行的产品标识、日期、净值、收益公式和格式；冷启动还要核对新表头，确认是否与你手工做的一致 |
| 7. 批准写入 | 执行写入：自动备份 → 写临时副本 → 校验 → 原子替换 | 先关闭正开着的正式表，然后明确对 AI 说"我已检查预览，同意写入" |
| 8. 定时预览（可选） | 安装 Windows 计划任务 | 是否安装由你决定；**定时任务只生成预览，永远不会自动写正式表** |

## 日常使用节奏

- **手动**：净值邮件到齐后，让 AI 跑一次预览 → 你检查 → 让 AI 写入。
- **定时**：任务自动出预览 → 你用 `navctl.py schedule status` 看最近一次成败（或
  翻 `logs\preview-日期.log`）→ 检查预览 → 手动让 AI 写入。
- **产品变动**：新增产品让 AI 加路由重新验证；某产品赎回或停更，让 AI 给该路由
  设置 `paused` 并注明原因，其余产品照常运行。

## 常见拦截与解决

| 看到的提示 | 原因 | 怎么办 |
|---|---|---|
| Windows 安装目录过长 | 路径超过 120 字符 | 换成 `D:\nav-runtime` 这类短路径 |
| Another runtime process is active | 上一次运行还没结束 | 等几分钟再试；程序崩溃后锁会自动释放，不需要手动删文件 |
| 无法替换正式工作簿……关闭窗口 | 正式表正被打开 | 关闭 Excel/WPS 中的正式表，重新写入 |
| latest routed NAV exceeds max_staleness_days | 该产品邮件长期未更新 | 核实是否赎回/停更；是则让 AI 设 `paused` |
| no routed NAV rows were found | 回看窗口内没有该发件人的净值邮件 | 确认发件人地址拼写、邮箱文件夹、回看天数 |
| summary row was not found | 路由处于默认严格模式，但工作表数据区后面没有“累计/合计”行 | 如果它本来就是简单追加表，让 AI 核实后显式配置 `sheet_mode: append`；复杂表不要为绕过错误切换模式 |
| historical value conflict | 邮件数据与表内历史对不上 | **不要放宽容差硬闯**；人工核实哪边正确后再处理 |
| 某发件人邮件解析不了 | 特殊格式 | 让 AI 在运行目录 `parsers\` 里写本地解析器，用历史邮件验证后再启用 |

## 夏普比率、最大回撤等专业指标

这套工具不会卡住后续分析。它先负责把“产品名称/代码 + 日期 + 净值”可靠写成基础序列；之后可以让 AI 在未托管的分析工作表或单独文件中计算夏普比率、最大回撤、年化收益、波动率等。

计算前请明确告诉 AI：使用单位净值还是累计净值、日频还是周频、年化因子、无风险利率、样本起止日期和缺失值处理。不要把这些自定义公式直接塞进受管汇总行；对分析页的修改也应单独生成预览并由你批准。

## 安全红线（记住这五条）

1. 授权码只通过 `secret set` 隐藏输入，**绝不**发给 AI、微信、邮件或任何聊天窗口。
2. 流程必须是"预览 → 你检查 → 你批准 → 写入"。哪个 AI 想跳过预览直接写，拒绝它。
3. 运行目录里有你的邮件数据、预览和备份——不要放进网盘同步，不要发给别人。
4. 每次写入前程序自动备份到 `backups\`，出了问题可以找回上一版。
5. 不再使用时先清理再删目录：
   `navctl.py schedule remove`（删定时任务）→ `navctl.py secret remove`（删密钥）。

## 命令速查（都在运行目录里执行）

```powershell
cd D:\nav-runtime
.\.venv\Scripts\python.exe navctl.py doctor            # 环境体检：四级就绪状态
.\.venv\Scripts\python.exe navctl.py preview           # 收邮件→核对→生成预览
.\.venv\Scripts\python.exe navctl.py commit --yes-reviewed-preview   # 检查预览后正式写入
.\.venv\Scripts\python.exe navctl.py schedule status   # 定时任务状态+最近一次结果
.\.venv\Scripts\python.exe navctl.py secret set        # 保存/更新邮箱授权码
```
