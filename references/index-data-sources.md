# 公共指数数据源参考

## 使用边界

当前运行时不会直接从公网抓取指数；它只读取已经在工作簿中核实的基准来源工作表。只有用户明确授权评估公共数据源时，才使用本参考调查并填充本地来源表。公开可访问不等于口径正确、接口稳定或允许商业使用。

选择顺序：指数编制方或交易所的正式渠道 > 已获授权的数据终端 > 经历史重叠验证的公开行情接口。未经核实的备用源不得静默补值，也不得覆盖已验证缓存。

## 2026-07-20 已复核的候选通道

### 中证指数官网

- 官网：[中证指数有限公司](https://www.csindex.com.cn/)
- 历史表现接口：`https://www.csindex.com.cn/csindex-home/perf/index-perf`
- 常用参数：`indexCode`、`startDate`、`endDate`，日期格式为 `YYYYMMDD`。
- 返回数据位于 `data`，常用字段为 `tradeDate` 和 `close`。

| 指数 | `indexCode` |
| --- | --- |
| 中证500 | `000905` |
| 中证1000 | `000852` |
| 中证2000 | `932000` |
| 中证全指 | `000985` |

优先使用该渠道核对中证系列的价格指数点位。接口可能晚于交易所收盘更新；不要把“请求成功但没有目标日”当作零收益。不要依赖 `endDate` 是否包含当天的隐含行为，应让请求范围覆盖目标日后再在本地按精确日期过滤。

### 腾讯行情兜底

- 日 K 接口：`https://web.ifzq.gtimg.cn/appstock/app/fqkline/get`
- 查询形态：`param={symbol},day,,,{count},qfq`
- 已取得历史数据的代码：中证500 `sh000905`、中证1000 `sh000852`、中证全指 `sh000985`。
- 返回日线通常位于 `data.<symbol>.day` 或 `qfqday`，日期和收盘价分别位于每行第 1、3 项。

该接口没有在本 Skill 中承诺的稳定版本或服务级别，只能作为缺日兜底。2026-07-20 的复核中，`sh932000` 没有返回中证2000日线，因此不能为它配置这个兜底。每次使用前都要检查返回代码、日期、数组结构和历史重叠值。

### 东方财富行情兜底

- 历史 K 线接口：`https://push2his.eastmoney.com/api/qt/stock/kline/get`
- 日线常用参数：`secid`、`fields1=f1`、`fields2=f51,f53`、`klt=101`、`fqt=0`、`beg`、`end`、`lmt`。
- 已取得历史数据的 `secid`：中证500 `1.000905`、中证1000 `1.000852`、中证2000 `2.932000`、北证50 `0.899050`。
- 返回日线通常位于 `data.klines`，在上述字段设置下每行形如 `日期,收盘价`。

该接口同样属于无稳定性承诺的网页行情通道。必须限制频率、设置超时与重试、保留最后一次已验证缓存，并防止较短的新响应删除缓存尾部。`push2delay` 的市场列表和逐股历史 K 线可以用于研究，但不能自动视为某个指数的官方点位。

### 通达信板块指数

通达信客户端口径中曾验证过板块指数 `880078`，上证红利可用代码 `000015`。这类代码属于特定行情终端的命名空间，只有目标工作簿明确沿用同一终端口径时才有意义。

不要在公开配置中固化行情服务器 IP。常见 Python 客户端 `pytdx` 的[上游仓库](https://github.com/rainx/pytdx)已经归档，作者明确说明项目停止维护并限制商业用途，因此不得把它作为公开 Skill 的默认企业数据源。需要该口径时，应优先使用用户单位已经获批的数据终端或授权接口，并由用户完成合规确认。

### 等权复现不是直接指数源

用某市场当前股票列表的当日涨跌幅做等权平均，只能得到近似复现值。成分历史、上市首日、停复牌、分红除权和再平衡细节都会造成长期漂移。除非指数方法、历史成分和复权规则均可证明，并已完成全历史核验，否则不得把复现链当作正式基准点位；最多作为带醒目警告的临时诊断值。

## 接入前验证清单

1. 让用户确认指数全名、代码、价格/全收益/净收益口径及目标工作簿原始来源。
2. 保存来源域名、原始代码、抓取时间和原始响应摘要；不得保存账号、令牌或真实工作簿内容到公开仓库。
3. 比较所有可重叠历史日期，至少覆盖 10 个非连续交易日和两个自然月；最新日期还要连续核对。
4. 分别按工作簿显示精度确定点位容差和收益容差，不得用同一个固定阈值掩盖口径错误。
5. 目标日缺失、代码不符、返回结构变化或出现无法解释的历史差异时，停止相关预览。
6. 多源合并时记录每个日期的实际来源；主源已有值时，备用源不得覆盖。
7. 请求范围应覆盖目标日期后再精确过滤，避免端点边界和发布时间差异造成缺日。
8. 缓存采用“已验证历史 + 新响应覆盖重叠日”的合并方式；短响应不得截断旧缓存。
9. 控制请求频率并遵守数据提供方条款；没有商业授权的库或接口不得用于企业正式流程。
10. 把最终核实的日期与点位写入本地基准来源工作表，再按 [configuration.md](configuration.md) 的 `benchmark` 规则映射；当前运行时不直接消费公网接口。
